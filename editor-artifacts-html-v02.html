Un destino propio para los HTML que salen del chat: hosteado, versionado en cada guardado, editable a mano en modo gráfico, y siempre al alcance de la IA para volver a tocarlo.

estado: en marcha ● versión: v0.2 ● autor: Iñaki ● guarda: commit real
§1 — El problema

La necesidad

Muchos outputs de interacciones complejas con el chat terminan siendo un documento HTML —con secciones e interacciones de distinto tipo— que hoy se comparte como adjunto suelto. Sin permisos, sin comentarios, sin link estable, ni forma cómoda de editarlo o que la IA lo retome.

La meta: darle a esos HTML un lugar donde vivan, con link, edición gráfica y round-trip con el chat.

§2 — Qué cumple

Requisitos

  • Hostear el HTML en un repositorio. ✓ GitHub.
  • Abrir con un link renderizado. ✓ htmlpreview.
  • Editar el texto en modo gráfico. ✓ toggle de modo edición.
  • Versionar: cada guardado, una versión. ✓ commit automático.
  • Round-trip con la IA: el chat lee y reedita. ✓ MCP de GitHub.
  • Permisos y comentarios para terceros. Por resolver.
§3 — La arquitectura

Tres piezas

Repo + htmlpreview hosting

Los artifacts viven como .html en ialganaras/html. htmlpreview los sirve renderizados sin configurar GitHub Pages.

Editor cliente contenteditable

Un script que prende contenteditable sobre las regiones marcadas con data-editable. Toggle de modo edición.

Cloudflare Worker commit

Un Worker con el token de GitHub como secreto. El botón "Guardar" le manda el HTML, el Worker hace PUT a la API de Contents: cada guardado, un commit nuevo.

§4 — El ciclo

Ida y vuelta

  1. Claude genera el HTML y lo commitea via MCP (o se sube a mano).
  2. Se abre por htmlpreview, se prende modo edición, se edita el texto.
  3. Guardar → Worker → commit nuevo en main.
  4. Para retomar, Claude lo lee del repo y reescribe.
  5. El historial queda en GitHub. Rollback gratis.
