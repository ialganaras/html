§1 — El problema

La necesidad

Muchos de los outputs de interacciones complejas con el chat terminan siendo un documento HTML —con secciones e interacciones de distinto tipo— que hoy se comparte como adjunto suelto. Eso significa: nada de permisos, nada de comentarios, nada de un link estable, y ninguna forma cómoda de editarlo después ni de que la IA lo retome.

La meta es darle a esos HTML un lugar donde vivan: que se suban a un repositorio, se abran con un link, se editen directo en modo gráfico, y que el chat los tenga a mano para leer y volver a editar.

§2 — Qué tiene que cumplir

Requisitos

  • Hostear el HTML en un repositorio, no como adjunto.
  • Abrir con un link renderizado, como una página viva.
  • Permisos sobre quién ve y quién edita.
  • Editar el texto en modo gráfico (WYSIWYG), sin tocar código.
  • Versionar: que cada guardado genere una versión nueva.
  • Round-trip con la IA: que el chat lo lea y lo reedite fácil.
§3 — La solución

El mini-build propio

En lugar de pagar un CMS de terceros, una arquitectura liviana de tres piezas, toda apoyada en GitHub. El HTML sigue siendo HTML y queda 100% bajo control propio.

Repo + Pages hosting

Los artifacts viven como archivos docs/*.html. GitHub Pages los sirve en una URL pública. El conector de GitHub de Claude los lee sin mover nada.

Editor cliente contenteditable

Un pequeño script que, en modo edición, prende contenteditable sobre las regiones marcadas y muestra un botón de guardar. Es lo que estás usando ahora mismo en este documento.

Proxy de commit serverless

Una función (Cloudflare Worker o val) con el token de GitHub como secreto. Al guardar, escribe el HTML vía la API de GitHub — y cada escritura es un commit, o sea una versión nueva con historial y rollback.

§4 — El ciclo

Ida y vuelta

  1. Claude genera el HTML en el chat.
  2. Se sube al repo de GitHub (este paso lo estamos probando ahora).
  3. Se abre y edita en modo gráfico; cada guardado deja un commit.
  4. Para retomarlo, se le pasa el link raw a Claude y lo lee de GitHub.
  5. Claude lo reedita y se vuelve a guardar. El círculo se cierra.
Este mismo archivo es la primera prueba del ciclo: nació en el chat y ahora va camino al repo.
§5 — Dónde estamos

Estado y próximos pasos

Primera etapa: confirmar que un HTML generado en el chat sube limpio a GitHub y vuelve legible para la IA. Después se suma el editor cliente con guardado real, y por último el proxy de commit para cerrar el versionado automático.
