Proyecto html-editor: el loop cerró

Snapshot del estado al 29-06-2026. Lo que está vivo, lo que faltó, y qué viene.

autor: Iñaki fecha: 29-06-2026 repo: ialganaras/html

§1 — ResumenQué resolvimos en esta vuelta

El proyecto arrancó esta sesión con el editor embebido dentro del HTML demo, el Worker desplegado en una versión vieja, sin Skill, sin endpoint MCP, y con el loop de escritura roto: yo podía leer del repo pero no escribir.

Después de cuatro vueltas de diagnóstico (algunas mías equivocadas), quedó todo el roadmap principal cerrado: el editor extraído como archivo standalone, el Worker como MCP server propio con auth, y el connector oficial de GitHub finalmente escribiendo en el repo.

§2 — EstadoLo que está vivo hoy

  • _editor.js standalone en la raíz del repo, autoinyecta toolbar + CSS, gate por ?edit=si, dirty-tracking por snapshot.
  • Worker html-commit v5 deployado en html-commit.inaki-4d6.workers.dev: serve sin cache con content-type por extensión, REST save/list/revert, y endpoint /mcp JSON-RPC con bearer auth.
  • MCP de GitHub oficial instalado como GitHub App con read+write a code/issues/PRs — escribe directo, sin pasar por el Worker.
  • Skill html-editable registrado en Claude Code; este mismo memo es la prueba de que la receta funciona end-to-end.

§3 — PendienteLo que queda en el roadmap

  • Comentarios embebidos (data-comments): UI para seleccionar región, comentar, serializar al atributo; los comentarios viajan con el archivo versionado.
  • Endurecer seguridad del Worker: el endpoint REST POST / sigue con CORS abierto y sin token; sumar origin allowlist o bearer compartido también ahí (el /mcp ya tiene auth).
  • Migrar el Skill a Claude.ai: replicar la receta como Custom Instructions de un Project, así también dispara en chats web/desktop, no solo en Claude Code.

§4 — AprendizajeEl diagnóstico equivocado

El loop de escritura tardó cuatro intentos porque arranqué con la hipótesis de que el plan free de GitHub no permitía escribir desde MCP. Era falsa. Después probé "no tiene scopes write en el OAuth"; también falsa. Después "el connector es read-only por diseño de Anthropic"; también falsa.

La verdad: el "Claude Github MCP Connector" es un GitHub App (no OAuth App), y los GitHub Apps necesitan dos cosas separadas — user authorization y repository installation. Solo había hecho la primera. Una vez instalado el app en el repo con permisos write, todo anduvo al primer intento.

Moraleja: cuando un 403 dice "Resource not accessible by integration", chequear si el client_id empieza con Iv23. Si sí, es GitHub App y falta installation, no scopes.
