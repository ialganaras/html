Propuesta de workflow · documento vivo

Artifacts HTML que van y vuelven de GitHub

Hosteado por el Worker, editable en modo gráfico, versionado en cada guardado, al alcance de la IA.

estado: en uso ● versión: v0.5 ● autor: Iñaki ● servido por: Cloudflare Worker
§1 — El problema

La necesidad

Muchos outputs de interacciones complejas con el chat terminan siendo un documento HTML que hoy se comparte como adjunto suelto. Sin permisos, sin comentarios, sin link estable, ni forma cómoda de editarlo o que la IA lo retome.

La meta: darle a esos HTML un lugar donde vivan, con link, edición gráfica y round-trip con el chat.

§2 — Qué cumple

Requisitos

  • Hostear el HTML en un repositorio. ✓ GitHub.
  • Abrir con un link renderizado. ✓ Worker propio (sin cache).
  • Editar el texto en modo gráfico. ✓ toggle de modo edición.
  • Versionar: cada guardado, una versión. ✓ commit automático.
  • Ver y restaurar versiones anteriores. ✓ panel de historial.
  • Saber si hay cambios sin guardar. ✓ indicador dirty.
  • Round-trip con la IA: el chat lee y reedita. ✓ MCP de GitHub.
  • Permisos y comentarios para terceros. Por resolver.
§3 — La arquitectura

Tres piezas

Repo de GitHub storage

Los artifacts viven como .html en ialganaras/html. GitHub es el storage y el versionado.

Editor cliente contenteditable

Script embebido que prende contenteditable sobre regiones marcadas con data-editable. Tracking de cambios, panel de historial.

Cloudflare Worker serve · save · list · revert

El Worker es quien sirve el HTML (vía la API de GitHub con cache 0) y quien recibe los commits. Eliminamos a htmlpreview del medio: el link es ahora directo al Worker.

§4 — El ciclo

Ida y vuelta

  1. Claude genera el HTML y lo commitea via MCP (o se sube a mano).
  2. Se abre por el Worker: https://html-commit.XXX.workers.dev/archivo.html.
  3. Se prende modo edición, se edita el texto. El LED naranja avisa cambios sin guardar.
  4. Guardar → Worker → commit nuevo. El indicador vuelve a verde.
  5. El link compartible muestra siempre lo último, sin cache.
